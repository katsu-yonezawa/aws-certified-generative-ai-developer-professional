# Bedrock モデルカスタマイズ 学習ノート

## 1. まず、LLM がどう育つか

LLM は、最初から質問応答が上手なわけではなく、いくつかの段階を経て使いやすいモデルになります。Amazon Nova の説明では、モデル開発には事前学習、その後の調整、指示追従や安全性のための整列といった段階があります。ここでは学習用に、かなりシンプルに整理します。 ([AWS ドキュメント](https://docs.aws.amazon.com/nova/latest/userguide/nova-forge-cpt.html?utm_source=chatgpt.com "Continued Pre-Training and Mid-Training - Amazon Nova"))

### ① 事前学習

大量の文章を読んで、言葉のつながり、一般知識、文章の型、よくある質問と回答の形などを広く身につける段階です。これが基盤モデルの土台になります。 ([AWS ドキュメント](https://docs.aws.amazon.com/nova/latest/userguide/nova-forge-cpt.html?utm_source=chatgpt.com "Continued Pre-Training and Mid-Training - Amazon Nova"))

### ② 必要に応じて、特定分野を追加で学ぶ

ここで出てくるのが Continued pre-training です。追加の未ラベル文書を読ませて、特定分野の知識、専門用語、文体、文書パターンに慣れさせます。 ([AWS ドキュメント](https://docs.aws.amazon.com/nova/latest/userguide/nova-cpt.html?utm_source=chatgpt.com "Continued pre-training (CPT) - Amazon Nova"))

### ③ 指示に従う形へ整える

ここで出てくるのが Fine-tuning です。入力と望ましい出力の組を使って、この入力ならこの形で返す、という振る舞いを学ばせます。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/custom-models.html?utm_source=chatgpt.com "Customize your model to improve its performance for ..."))

### ④ 評価基準に沿って、さらに答え方を磨く

ここで出てくるのが Reinforcement fine-tuning です。正解文そのものよりも、採点結果を使って、より望ましい答えを出しやすくする方向へ学習させます。Bedrock では、各プロンプトに対して複数の応答を生成し、reward function で評価して学習します。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reinforcement-fine-tuning.html?utm_source=chatgpt.com "Customize a model with reinforcement fine-tuning in ..."))

### ⑤ 本番で使う

学習が終わったモデルを推論で使います。なお、RAG や Knowledge Bases は、学習そのものではなく、本番時に外部知識を引いて答える仕組みです。モデルのカスタマイズとは別物として切り分けると整理しやすいです。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/custom-models.html?utm_source=chatgpt.com "Customize your model to improve its performance for ..."))

## 2. 4つの手法を一気につなげて理解する

### Fine-tuning

**何を渡すか**  
入力と、望ましい出力のペアです。たとえば prompt と completion、あるいは会話形式のデータです。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/custom-models.html?utm_source=chatgpt.com "Customize your model to improve its performance for ..."))

**何を学ばせるか**  
出力形式、応答パターン、口調、分類ラベル、JSON の形などです。つまり、期待する返し方を覚えさせます。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/custom-models.html?utm_source=chatgpt.com "Customize your model to improve its performance for ..."))

**向いている場面**  
問い合わせ分類、要約フォーマット固定、定型応答、決まった JSON 出力などです。正解例を素直に作れる仕事と相性がよいです。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/custom-models.html?utm_source=chatgpt.com "Customize your model to improve its performance for ..."))

---

### Continued pre-training

**何を渡すか**  
ラベルなしの生文書です。Bedrock のデータ準備では、各行が `input` だけを持つ JSONL 形式です。 ([AWS ドキュメント](https://docs.aws.amazon.com/ja_jp/bedrock/latest/userguide/dataset-prep-continued-pretraining.html?utm_source=chatgpt.com "継続的な事前トレーニング用にデータセットを準備する"))

**何を学ばせるか**  
分野知識、専門用語、文章の癖、ドメイン特有の表現です。Fine-tuning が答え方を整えるものだとすると、こちらは知識と文脈への慣れを深める方向です。 ([AWS ドキュメント](https://docs.aws.amazon.com/nova/latest/userguide/nova-cpt.html?utm_source=chatgpt.com "Continued pre-training (CPT) - Amazon Nova"))

**向いている場面**  
医療、金融、法務、製造、社内文書のように、大量の文書はあるけれど、質問と模範回答のペアは少ない場合です。 ([AWS ドキュメント](https://docs.aws.amazon.com/nova/latest/userguide/nova-cpt.html?utm_source=chatgpt.com "Continued pre-training (CPT) - Amazon Nova"))

---

### Distillation

**何を渡すか**  
教師モデルに投げるための入力プロンプトです。Bedrock は教師モデルの応答を生成し、その応答を使って生徒モデルを fine-tune します。本番の invocation logs を使う方法もあります。 ([AWS ドキュメント](https://docs.aws.amazon.com/ja_jp/bedrock/latest/userguide/model-distillation.html?utm_source=chatgpt.com "Amazon Bedrock で蒸留を使用してモデルをカスタマイズする"))

**何を学ばせるか**  
大きく高性能なモデルの振る舞いを、より小さく軽いモデルへ移します。主眼は知識追加というより、品質と速度とコストのバランスを整えることです。 ([AWS ドキュメント](https://docs.aws.amazon.com/ja_jp/bedrock/latest/userguide/model-distillation.html?utm_source=chatgpt.com "Amazon Bedrock で蒸留を使用してモデルをカスタマイズする"))

**向いている場面**  
高性能モデルの応答品質に近づけたいけれど、本番ではもっと小さいモデルを使いたい場合です。 ([AWS ドキュメント](https://docs.aws.amazon.com/ja_jp/bedrock/latest/userguide/model-distillation.html?utm_source=chatgpt.com "Amazon Bedrock で蒸留を使用してモデルをカスタマイズする"))

---

### Reinforcement fine-tuning

**何を渡すか**  
主にプロンプトと、採点の仕組みです。Bedrock の reward function は、Lambda で独自採点ロジックを書く方法と、モデルを judge として使う方法があります。instruction following や format validation などのテンプレートもあります。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reward-functions.html?utm_source=chatgpt.com "Setting up reward functions for Amazon Nova models"))

**何を学ばせるか**  
正解文を丸暗記させるのではなく、評価基準に沿ってよりよい答えを出す方向へ調整します。Bedrock は複数応答を採点し、そのスコアをもとに学習します。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reinforcement-fine-tuning.html?utm_source=chatgpt.com "Customize a model with reinforcement fine-tuning in ..."))

**向いている場面**  
正解文は1つに決めにくいが、良し悪しは採点できる仕事です。AWS の例では、コード生成、数理問題、指示追従、要約、RAG の忠実性評価、コンテンツモデレーションなどが挙げられています。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reinforcement-fine-tuning.html?utm_source=chatgpt.com "Customize a model with reinforcement fine-tuning in ..."))

## 3. 4つの違いを最短で見分ける

- **Fine-tuning**  
  正解例を見せて、期待する返し方を覚えさせる。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/custom-models.html?utm_source=chatgpt.com "Customize your model to improve its performance for ..."))

- **Continued pre-training**  
  生文書を追加で読ませて、その分野に強くする。 ([AWS ドキュメント](https://docs.aws.amazon.com/nova/latest/userguide/nova-cpt.html?utm_source=chatgpt.com "Continued pre-training (CPT) - Amazon Nova"))

- **Distillation**  
  高性能モデルの応答を使って、小さいモデルへ能力を移す。 ([AWS ドキュメント](https://docs.aws.amazon.com/ja_jp/bedrock/latest/userguide/model-distillation.html?utm_source=chatgpt.com "Amazon Bedrock で蒸留を使用してモデルをカスタマイズする"))

- **Reinforcement fine-tuning**  
  採点基準を使って、より望ましい答え方へ磨く。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reinforcement-fine-tuning.html?utm_source=chatgpt.com "Customize a model with reinforcement fine-tuning in ..."))

## 4. Reinforcement fine-tuning が向く具体例

### コード生成

同じ要件を満たすコードは何通りもあります。模範解答を1本に決めるより、テストが通るか、禁止 API を使っていないか、性能条件を満たすかで採点するほうが自然です。AWS も、実行結果などで客観評価しやすいタスクとしてコード生成を挙げています。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reinforcement-fine-tuning.html?utm_source=chatgpt.com "Customize a model with reinforcement fine-tuning in ..."))

### SQL 生成

同じ結果を返す SQL は複数ありえます。期待する結果セットになるか、構文エラーがないか、制約違反がないかで採点できます。これは programmatically verifiable なタスクの典型です。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reinforcement-fine-tuning.html?utm_source=chatgpt.com "Customize a model with reinforcement fine-tuning in ..."))

### 要約

よい要約は1つではありません。しかし、重要論点を含むか、長さ制約を守るか、原文にない内容を足していないかは評価できます。AWS も要約を AI judge が有効な主観的タスクとして挙げています。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reward-functions.html?utm_source=chatgpt.com "Setting up reward functions for Amazon Nova models"))

### 指示追従

社内アシスタントで、丁寧語、禁止事項順守、指定フォーマット、長さ制約などを守らせたい場合、理想の答えは複数ありますが、ルール順守の度合いは採点できます。instruction following は Bedrock の reward function テンプレート対象です。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reward-functions.html?utm_source=chatgpt.com "Setting up reward functions for Amazon Nova models"))

### RAG の忠実性

RAG では、取得した文脈に沿って答えているか、文脈にないことを言っていないか、必要な引用方針を守っているかを評価したい場面があります。これも模範解答より、評価基準のほうが作りやすい例です。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reward-functions.html?utm_source=chatgpt.com "Setting up reward functions for Amazon Nova models"))

## 5. どれを選ぶかの考え方

### Fine-tuning を選びやすいとき

正解の出力例をたくさん用意できるときです。分類、変換、固定フォーマット出力のような仕事に向きます。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/custom-models.html?utm_source=chatgpt.com "Customize your model to improve its performance for ..."))

### Continued pre-training を選びやすいとき

大量のドメイン文書はあるが、ラベル付きデータは少ないときです。専門分野への慣れを深めたい場合に向きます。 ([AWS ドキュメント](https://docs.aws.amazon.com/nova/latest/userguide/nova-cpt.html?utm_source=chatgpt.com "Continued pre-training (CPT) - Amazon Nova"))

### Distillation を選びやすいとき

品質をできるだけ保ちつつ、より小さいモデルで速く安く回したいときです。 ([AWS ドキュメント](https://docs.aws.amazon.com/ja_jp/bedrock/latest/userguide/model-distillation.html?utm_source=chatgpt.com "Amazon Bedrock で蒸留を使用してモデルをカスタマイズする"))

### Reinforcement fine-tuning を選びやすいとき

正解文は書きにくいが、採点基準なら作れるときです。とくに、品質・安全性・形式順守・指示追従のような多面的な評価をしたい場面に向きます。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reinforcement-fine-tuning.html?utm_source=chatgpt.com "Customize a model with reinforcement fine-tuning in ..."))

## 6. 試験対策用の覚え方

覚えるときは、**学習データの形**で切ると見分けやすいです。

- 文書だけを読む → **Continued pre-training** ([AWS ドキュメント](https://docs.aws.amazon.com/ja_jp/bedrock/latest/userguide/dataset-prep-continued-pretraining.html?utm_source=chatgpt.com "継続的な事前トレーニング用にデータセットを準備する"))

- 入力と正解出力のペア → **Fine-tuning** ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/custom-models.html?utm_source=chatgpt.com "Customize your model to improve its performance for ..."))

- 教師モデルの応答を使う → **Distillation** ([AWS ドキュメント](https://docs.aws.amazon.com/ja_jp/bedrock/latest/userguide/model-distillation.html?utm_source=chatgpt.com "Amazon Bedrock で蒸留を使用してモデルをカスタマイズする"))

- 採点関数で良し悪しを見る → **Reinforcement fine-tuning** ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/reinforcement-fine-tuning.html?utm_source=chatgpt.com "Customize a model with reinforcement fine-tuning in ..."))

## 7. 最後に、ひとことで整理

このテーマは、**モデルに何を食べさせて、何を伸ばすのか**で整理すると混ざりにくくなります。  
知識を厚くしたいのか、返し方をそろえたいのか、先生モデルの品質を移したいのか、採点基準で答え方を磨きたいのか。この4つを切り分けられると、Bedrock のモデルカスタマイズはかなり見やすくなります。 ([AWS ドキュメント](https://docs.aws.amazon.com/bedrock/latest/userguide/custom-models.html?utm_source=chatgpt.com "Customize your model to improve its performance for ..."))

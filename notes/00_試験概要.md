# AWS Certified Generative AI Developer - Professional 試験概要

## ひとことで

AWS 上で生成 AI を本番実装する開発者向けの上位試験。  
問われるのはモデル理論より、`FM統合 + RAG + エージェント + セキュリティ + 運用最適化 + 評価` の実装判断。

## この試験が証明すること

- 基盤モデルを業務アプリに安全に統合できる
- RAG、ベクトル検索、プロンプト管理を設計できる
- エージェントやツール連携を実装できる
- コスト、性能、可観測性を本番水準で最適化できる
- セキュリティ、プライバシー、Responsible AI を組み込める
- 評価、監視、障害切り分けまで含めて運用できる

## 想定受験者

- AWS またはオープンソースで本番アプリを構築した経験が 2 年以上
- AI/ML またはデータエンジニアリングの実務経験がある
- GenAI ソリューション実装の実務経験が 1 年ある

前提として欲しい知識:

- AWS のコンピューティング、ストレージ、ネットワーク
- IAM、暗号化、監査
- API 開発、IaC、CI/CD
- 監視、コスト最適化

## 対象外

- モデル開発と大規模トレーニング
- 高度な ML 理論
- 本格的な特徴量エンジニアリング

要点:

- この試験は `モデルを作る人` より `既存 FM を安全に組み込む人` 向け
- ただし SageMaker AI の学習、微調整、監視、ガバナンスは周辺知識として重要

## 試験形式

2026-03-15 時点の整理:

- グローバルの認定ページは `180 minutes`、`75 questions`、`300 USD`、`English/Japanese/Korean/Simplified Chinese` を案内
- 試験ガイドは `採点対象 65 問 + 採点対象外 10 問`、`100-1000 点`、`合格 750`、`未回答は不正解`、`推測による減点なし` を案内
- 日本語の認定ページには、まだ `Beta` の `205 minutes`、`85 questions`、`150 USD` が残っている

受験前の扱い:

- 学習時は試験ガイドを基準にする
- 申込条件は Pearson VUE と最新の認定ページで再確認する

## 出題ドメイン

### 1. 基盤モデルの統合、データ管理、コンプライアンス `31%`

- FM 選定
- データ前処理、マルチモーダル入力
- 埋め込み、ベクトルストア、チャンク分割
- RAG、再ランキング
- プロンプト管理、データ由来、コンプライアンス

### 2. 実装と統合 `26%`

- エージェンティック AI
- ツール連携、状態管理、メモリ
- モデル配備戦略
- FM API 統合、ストリーミング、リトライ
- API Gateway、Lambda、Step Functions、EventBridge などの実装パターン

### 3. AI の安全性、セキュリティ、ガバナンス `20%`

- 入出力の安全制御
- ハルシネーション低減
- prompt injection / jailbreak 対策
- PII、プライバシー、データ保持
- IAM、VPC、監査、Responsible AI

### 4. 運用効率と最適化 `12%`

- トークン効率
- キャッシュ
- レイテンシ最適化
- 容量計画、スループット
- 監視、可観測性、コスト管理

### 5. テスト、検証、トラブルシューティング `11%`

- FM 出力評価
- RAG 評価
- A/B テスト、カナリア
- 障害解析
- プロンプト、検索、統合の不具合切り分け

### 最優先

- `Amazon Bedrock`
- `Knowledge Bases`

- `Prompt management`
- `RAG の設計要素`

### 最後に詰める

- `Model Cards`、`Lineage`、`Clarify`、`Model Monitor`

## 補助ノートの位置づけ

- `08_生成AIの基礎整理` は試験の直接出題を増やすためというより、設問の判断を安定させる補助ノート
- とくに `RAG と fine-tuning の違い`、`temperature の意味`、`prompt より context と評価が重要` の整理に効く
- 逆に、内部アーキテクチャの深掘りだけを目的に時間を使いすぎない

## 合格向けの学習順

1. `00_試験概要` で全体像をつかむ
2. `08_生成AIの基礎整理` を読み、推論パラメータや prompt の前提をそろえる
3. `02_Bedrock` と `01_SageMaker` で主要サービスを整理する
4. `03_実装と統合` と `04_安全性とガバナンス` で Domain 2 と 3 を固める
5. `05_運用最適化と監視` と `06_評価とトラブルシューティング` で Domain 4 と 5 を固める
6. `07_頻出比較と解答パターン` を反復し、比較問題に強くする

## 頻出の判断軸

- `どの FM / API / 構成を選ぶか`
- `RAG で解くか、モデルを調整するか`
- `Amazon Q Business` で早く提供するか、Bedrock で自作するか
- `Agent にするか、固定ワークフローにするか`
- `Guardrails / IAM / VPC / 監査をどう組むか`
- `品質 / コスト / レイテンシのトレードオフをどう取るか`

## 問題文の読み方

### 先に拾うべき3種類

- `目的`
  何を実現したいのか。検索、生成、実行、評価、監査のどれが主役かを見る
- `制約`
  低レイテンシ、低コスト、閉域、権限制御、即時導入など、答えを狭める条件を見る
- `リスク`
  PII、誤回答、高リスク更新、規制対応など、追加で必要な安全策を見る

要点:

- 試験は `使えるサービスを全部言う` 問題ではなく、`主役の要件に最も自然な第一候補` を選ぶ問題が多い
- 同じ要件でも、`早く出したい` があると `Amazon Q Business` や `Bedrock` が強くなり、`深く制御したい` があると `SageMaker AI` や `Step Functions` が強くなる
- `動く` だけでなく `安全に運用できるか` まで見ると誤答を外しやすい

### 消去法の当て方

1. 主役機能を1つ決める
2. 監視、監査、セキュリティなどの補助機能を分けて考える
3. 問題文が求める最小構成を超える選択肢を落とす
4. 生成AI固有の原因を先に疑う

補足:

- `最新知識` はまず `RAG`
- `出力形式の安定` はまず `structured outputs`
- `高リスク操作` はまず `human review`
- `断続利用の推論` はまず `Serverless`
- `実行時の安全制御` はまず `Guardrails`

## Active Recall

- この試験は何を証明する試験か
- 最も配点が高いドメインは何か
- 試験ガイド上の採点対象問題数は何問か
- 合否はドメイン単位か、総合点か
- RAG の主要設計要素は何か
- Agent を本番運用するうえで必要な安全策は何か
- 複数選択問題で空欄を残す合理性はあるか

### 答えと解説

1. 既存の基盤モデルを AWS 上で安全に本番統合し、運用まで回せることを証明する試験です。中心は実装判断です。
2. 最も配点が高いのは Domain 1 で `31%` です。FM 選定、RAG、ベクトル検索、プロンプト管理が中心です。
3. 試験ガイド上は `採点対象 65 問`、`採点対象外 10 問` です。受験予約時の表示とは分けて覚えます。
4. 合否はドメイン別の足切りではなく総合点です。配点の高い Domain 1 と 2 を先に固めるのが効率的です。
5. 埋め込み、ベクトルストア、チャンク分割、検索、再ランキング、メタデータ設計です。RAG は検索設計が品質を左右します。
6. 入出力フィルタ、権限制御、監査、PII 対策、prompt injection 対策です。`動く` だけでは足りません。
7. ありません。未回答は不正解で、推測による減点もありません。消去法でも必ず埋める前提です。
